КРИТЕРИИ РУЧНОГО СКАЧИВАНИЯ ФИНАНСОВЫХ ОТЧЕТОВ

Задача
=====

Нужно скачать финансовые отчеты эмитентов для point-in-time слоя проекта.
Модель принимает решения по месячным срезам с 2023-07-01 по 2025-08-31.
Отчет можно использовать только с даты его публикации, поэтому главный критерий:

    publish_date <= decision_date

Для безопасного покрытия текущего модельного окна скачиваем раскрытия,
опубликованные с 2022-01-01 по 2025-08-31 включительно.


Какие годы и периоды нужны
==========================

Минимальный набор:

1. Годовая отчетность / audited FY:
   - FY 2021, если опубликована в 2022 году;
   - FY 2022;
   - FY 2023;
   - FY 2024.

2. Промежуточная МСФО / IFRS, если доступна:
   - Q1 2022, H1 2022, 9M 2022;
   - Q1 2023, H1 2023, 9M 2023;
   - Q1 2024, H1 2024, 9M 2024;
   - Q1 2025 и H1 2025, только если опубликованы не позднее 2025-08-31.

3. FY 2025 для текущего дипломного окна не нужен, потому что обычно публикуется
   в 2026 году и не мог быть доступен на 2025-08-31.


Какие типы документов скачивать
===============================

Приоритет 1: консолидированная финансовая отчетность МСФО / IFRS.
Примеры названий:
    - Consolidated Financial Statements
    - IFRS Financial Statements
    - Interim Condensed Consolidated Financial Statements
    - Финансовая отчетность по МСФО
    - Консолидированная финансовая отчетность

Приоритет 2: годовой отчет / annual report.
Он нужен для текстовых risk signals: санкции, валютный риск, ликвидность,
ковенанты, судебные риски, аудит, дивидендные ограничения, capex pressure.

Приоритет 3: РСБУ / standalone statements.
Скачивать только если МСФО за период нет или компания раскрывает только РСБУ.

Приоритет 4: презентации и press releases.
Использовать только как fallback, если нет полноценной отчетности. Презентация
хуже для модели, потому что часто содержит меньше примечаний и audit text.


Что НЕ нужно
============

Не скачивать как основной источник:

    - ESG / sustainability reports;
    - уставы, проспекты, сообщения о существенных фактах;
    - отдельные дивидендные решения;
    - новости без таблиц финансовых результатов;
    - презентации, если рядом есть полноценная МСФО-отчетность;
    - документы, опубликованные после 2025-08-31.


Форматы файлов
==============

Поддерживаются:

    .pdf
    .xlsx
    .xlsm
    .zip
    .txt
    .html

Если есть PDF и XLSX за один и тот же период, лучше скачать оба.
PDF полезен для текста и рисков, XLSX полезен для таблиц.

Если e-disclosure скачивает ZIP, оставляй ZIP как есть: extractor умеет читать
файлы внутри архива.


Куда класть файлы
=================

Клади документы в папки по тикерам:

    data/raw/reports/SBER/
    data/raw/reports/GAZP/
    data/raw/reports/LKOH/
    ...

Рекомендуемый формат имени файла:

    TICKER_reportPeriodEnd_publishDate_type.ext

Примеры:

    SBER_2024-12-31_2025-03-01_ifrs.pdf
    GAZP_2023-12-31_2024-04-29_annual_report.pdf
    LKOH_2025-06-30_2025-08-28_ifrs_h1.pdf

Если точная дата публикации неизвестна, используй:

    TICKER_reportPeriodEnd_unknown_type.ext

Но лучше потом восстановить publish_date по странице раскрытия, пресс-релизу
или дате публикации файла на сайте эмитента.


Какие поля потом нужны для report_registry.csv
==============================================

Для каждого скачанного файла нужно будет заполнить:

    ticker              тикер из модели: SBER, GAZP, LKOH и т.д.
    company_name        название компании
    report_period_end   дата конца отчетного периода, например 2024-12-31
    publish_date        дата публикации отчета, например 2025-03-01
    report_type         ifrs, ifrs_q1, ifrs_h1, ifrs_9m, annual_report, ras
    accounting_standard IFRS, RAS или пусто
    language            ru, en или mixed
    source_url          ссылка, откуда скачан файл
    local_path          путь относительно data/raw/reports, например SBER/SBER_2024-12-31_2025-03-01_ifrs.pdf

Самое важное поле - publish_date. Без него отчет не будет корректно
присоединяться point-in-time.


Особые замечания
================

Банки и финансовые компании:
    Для SBER, VTBR, MOEX важны банковские строки: net interest income,
    assets, equity, capital ratios, provisions, profit, liquidity/capital risks.

Сырьевые и промышленные компании:
    Для GAZP, LKOH, ROSN, NVTK, GMKN, TATN, CHMF, PLZL, ALRS, PHOR, RUAL, SNGS
    особенно важны debt, EBITDA, capex, free cash flow, FX/sanctions/capex risks.

YNDX:
    В текущем списке это legacy Yandex NV. Не смешивать старый YNDX и текущий
    российский Яндекс/YDEX без отдельного решения по universe.
